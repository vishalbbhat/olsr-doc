◆ BASEPATH_LENGTH

#define BASEPATH_LENGTH   0

these macros should be used to generate OONF logging output the OONF_severity_NH() variants don't print the timestamp/file/line header, which can be used for multiline output.

OONF_DEBUG should be used for all output that is only usefull for debugging a specific part of the code. This could be information about the internal progress of a function, state of variables, ...

OONF_INFO should be used for all output that does not inform the user about a problem/error in OONF. Examples would be "SPF run triggered" or "Hello package received from XXX.XXX.XXX.XXX".

OONF_WARN should be used for all error messages.
